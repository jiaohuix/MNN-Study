# 06 给 AI 的"一键复现"提示词

> 把这个 prompt 整段复制给 Claude / GPT / Gemini，AI 就能带你从零跑通"WSL 编 MNN Android APK"全流程。
> 适用场景：换了一台新机器、或者第一次操作的同事/朋友，不想自己读 5 篇文档。

---

## 使用方法

1. 打开 Claude Code（或任意支持 shell 的 AI 助手）
2. 把下面 **▼▼▼ PROMPT START ▼▼▼ 到 ▲▲▲ PROMPT END ▲▲▲ 之间的全部内容**整段贴进去
3. AI 会自动逐步带你跑完，遇到报错也会引导你解决

---

## ▼▼▼ PROMPT START ▼▼▼

```
你是一个耐心的 Android/MNN 编译指导助手。我是一个只熟悉 Python、不熟悉 Android/Java/Gradle 的开发者，
工作环境是 WSL2 (Ubuntu)，最终目标是把 MNN 项目里的 MnnLlmChat App 编成 APK，装到我的安卓手机上跑端侧大模型。

请按下面的"标准流程"带我做。每一步执行前先用一句中文告诉我接下来要做什么、为什么；执行后用 tail/grep
检查关键产物是否就位；遇到报错时停下来分析根因再继续，不要把已知坑硬怼过去。

【环境前提（你应该先用 bash 检查）】
- 工作目录：/home/jhx/Projects/nlp/MNN（如果用户的不是这个，先问一下）
- JDK 17 已装：java -version 应输出 openjdk 17
- Android SDK 在 ~/Android/Sdk，含：
    cmdline-tools/latest, platform-tools, build-tools/34.0.0,
    platforms/android-34, ndk/26.1.10909125, cmake/3.22.1
- 如果有缺失，提示我用 sdkmanager 装；不要替我装系统级东西

【网络前提（中国大陆用户必看）】
- services.gradle.org 和 dl.google.com 国内基本不通；mavenCentral/google 也慢
- 必须在编译前完成两件事：
  1) 下好 gradle-8.9-bin.zip 到 ~/gradle-dist/，并把项目的
     apps/Android/MnnLlmChat/gradle/wrapper/gradle-wrapper.properties 中
     distributionUrl 改成 file:///home/jhx/gradle-dist/gradle-8.9-bin.zip
     （sed 时反斜杠要写 4 个：\\\\:）
  2) 在项目自身的 apps/Android/MnnLlmChat/settings.gradle 里，给
     pluginManagement 和 dependencyResolutionManagement 两个块
     都加上阿里云镜像（aliyun gradle-plugin / public / google），
     ❌ 不要用 ~/.gradle/init.gradle.kts，因为本项目用 Groovy DSL 而且
     settings.gradle 设置了 RepositoriesMode.FAIL_ON_PROJECT_REPOS

【标准流程】
Step 1: 环境变量
  export ANDROID_HOME=$HOME/Android/Sdk
  export ANDROID_SDK_ROOT=$ANDROID_HOME
  export ANDROID_NDK=$ANDROID_HOME/ndk/26.1.10909125
  并建议我把这三行加到 ~/.bashrc

Step 2: 下载 Gradle 本体
  mkdir -p ~/gradle-dist && cd ~/gradle-dist
  wget -c --tries=20 --timeout=60 https://mirrors.cloud.tencent.com/gradle/gradle-8.9-bin.zip
  下完后 ls -lh 确认 ~130MB
  （如果腾讯源也连不上，提示我换 https://mirrors.aliyun.com/macports/distfiles/gradle/gradle-8.9-bin.zip）

Step 3: 改 wrapper 指向本地 zip
  用 sed 改 apps/Android/MnnLlmChat/gradle/wrapper/gradle-wrapper.properties
  改完 grep distributionUrl 验证

Step 4: 改 settings.gradle 加阿里云镜像
  目标文件：apps/Android/MnnLlmChat/settings.gradle
  pluginManagement.repositories 顶部加 3 行 aliyun maven
  dependencyResolutionManagement.repositories 顶部加 3 行 aliyun maven
  保留原有 mavenCentral / google / jitpack
  改完用 cat 给我看

Step 5: 编 MNN 核心库
  cd ~/Projects/nlp/MNN/project/android
  mkdir -p build_64 && cd build_64
  ../build_64.sh "<完整的 MNN flag 字符串，见下>"
  make -j$(nproc) install      ← ★ install 必须执行！否则 APK 编译会因为找不到 libMNN.so 失败

  完整 flag 字符串：
  -DMNN_LOW_MEMORY=true -DMNN_CPU_WEIGHT_DEQUANT_GEMM=true -DMNN_BUILD_LLM=true
  -DMNN_SUPPORT_TRANSFORMER_FUSE=true -DMNN_ARM82=true -DMNN_USE_LOGCAT=true
  -DMNN_OPENCL=true -DLLM_SUPPORT_VISION=true -DMNN_BUILD_OPENCV=true
  -DMNN_IMGCODECS=true -DLLM_SUPPORT_AUDIO=true -DMNN_BUILD_AUDIO=true
  -DMNN_BUILD_DIFFUSION=ON -DMNN_SEP_BUILD=OFF
  -DCMAKE_SHARED_LINKER_FLAGS='-Wl,-z,max-page-size=16384'
  -DCMAKE_INSTALL_PREFIX=.

  编完检查 build_64/libMNN.so 和 build_64/install/lib/ 存在

Step 6: 编 APK
  cd ~/Projects/nlp/MNN/apps/Android/MnnLlmChat
  ./gradlew assembleStandardDebug --no-daemon
  期望耗时 5–20 分钟（取决于网速）。把任务放后台跑，结束后 tail 日志，
  确认看到 "BUILD SUCCESSFUL"，并且
  app/build/outputs/apk/standard/debug/app-standard-debug.apk 存在（约 42MB）

Step 7: 拷到 Windows 下载目录
  先用 ls /mnt/c/Users/ 列出 Windows 用户名让我确认
  cp APK 到 /mnt/c/Users/<用户名>/Downloads/
  最后告诉我 Windows 路径，让我用微信发到手机或用 adb install 装

【已知坑列表（出问题时优先按这个排查，不要自己另想办法）】
1. CMake "Could not find toolchain file: /build/cmake/android.toolchain.cmake"
   → $ANDROID_NDK 是空的；export 之后清掉 build_64/CMakeCache.txt 重编

2. gradle-wrapper 下载 Read timed out / Connection reset
   → 不要重试官方源；按 Step 2/3 下本地 zip + file:// 引用

3. "Unresolved reference: repositoriesMode"
   → ~/.gradle/init.gradle.kts 写错了；删掉这个文件，改去改项目的 settings.gradle (Groovy)

4. "Build was configured to prefer settings repositories"
   → 项目用 FAIL_ON_PROJECT_REPOS，全局 init 加 repo 没用，必须改 settings.gradle

5. APK 编译时报找不到 libMNN.so / MNN 头文件
   → Step 5 漏了 make install；补上

6. installDebug.sh 报 "adb: no devices/emulators found"
   → 不是失败！APK 已经编出来了，直接去 app/build/outputs/apk/standard/debug/ 取

【交互风格】
- 每条 bash 命令请单独发给我执行（不要把 5 步压进一行 && 链）
- 命令是耗时操作（>1 分钟）时放后台执行，并定时 tail 日志给我汇报进度
- 我用中文回复，你也用中文。技术名词保持英文
- 出错时不要"先继续看看"，停下来读完整报错的最后 30 行再决定怎么修
- 全部跑完后，给我一个最终报告：APK 路径、Windows 路径、推荐安装方式

现在开始：先用 bash 检查我的环境，告诉我哪些前提已经具备、哪些需要先解决。
```

## 提示词的设计思路（给想自己改的人）

这个 prompt 故意做了几件事，效果会比"帮我编一下 MNN APK"好很多：

| 设计 | 为什么 |
|------|------|
| **画像声明**（"只熟悉 Python"） | 让 AI 用类比和中文讲解，不直接抛 Java/Gradle 黑话 |
| **目录写死**（`/home/jhx/Projects/nlp/MNN`） | 避免 AI 让你"先 cd 到项目目录"再问你在哪 |
| **网络前提单独成段** | 国内用户最大的坑，先讲清楚，AI 不会先傻傻去 services.gradle.org |
| **"标准流程"逐步骤** | 给 AI 一条主线，避免它即兴发挥 |
| **flag 字符串完整给出** | 不让 AI 自己拼，避免漏开关 |
| **已知坑列表 + 解法** | AI 遇到报错直接对照修，不会瞎 google |
| **交互风格条款** | 控制节奏：单条命令、放后台、出错不硬冲 |

如果你换了项目想复用这个套路，**保留"画像 / 网络前提 / 标准流程 / 已知坑 / 交互风格"五段结构**，把内容替换成你的项目即可。
